# 📖 Data Catalog

## Overview

The **Gold Layer** contains business-ready datasets designed for reporting, analytics, and dashboarding. These tables are created from validated **Silver Layer** tables and provide aggregated business metrics for healthcare operations.

| Property | Value |
|----------|-------|
| **Project** | Healthcare Data Engineering Pipeline |
| **Platform** | Databricks Lakehouse |
| **Architecture** | Medallion Architecture (Bronze → Silver → Gold) |
| **Storage Format** | Delta Tables |
| **Layer Documented** | Gold |
| **Refresh Strategy** | Full Refresh (Overwrite) |

---

# Gold Layer Tables

| Table | Business Purpose |
|--------|------------------|
| `gold.revenue_by_hospital` | Analyze revenue generated by each hospital |

# gold.revenue_by_hospital

## Purpose

Provides the total revenue generated by each hospital. This dataset supports financial reporting and hospital performance analysis.

### Source Tables

- `silver.billing`
- `silver.admissions`
- `silver.hospitals`

### Columns

| Column | Data Type | Description |
|---------|-----------|-------------|
| hospital_id | INT | Unique identifier for each hospital |
| hospital_name | STRING | Hospital name |
| state | STRING | Hospital location |
| total_revenue | DOUBLE | Total revenue generated by the hospital |

### Business Use Cases

- Executive dashboards
- Hospital performance analysis
- Financial reporting

# gold.procedure_performance

## Purpose

Measures how frequently each medical procedure is performed across the healthcare system.

### Source Tables

- `silver.admissions`
- `silver.procedures`

### Columns

| Column | Data Type | Description |
|---------|-----------|-------------|
| procedure_id | INT | Unique procedure identifier |
| procedure_name | STRING | Medical procedure name |
| department | STRING | Clinical department performing the procedure |
| procedure_cost | DOUBLE | Standard cost of the procedure |
| times_performed | BIGINT | Number of times the procedure was performed |

### Business Use Cases

- Procedure utilization analysis
- Department workload analysis
- Clinical operations reporting

# Notes

- All Gold tables are created from validated Silver layer tables.
- Data is stored as Delta Tables in the Databricks Lakehouse.
- Gold tables are refreshed using a **Full Refresh (Overwrite)** strategy.
- These datasets are optimized for analytics and dashboarding rather than transactional processing.
